# Android Visual Testing App

This Android application provides a visual testing environment for the Compose output generated by the Style Converter.

## Features

- Reads `composeOutput.json` from the assets folder
- Displays all components in a scrollable list
- Shows base modifiers, states, and responsive variants for each component
- Displays the generated Compose code in a code viewer

## Setup

1. **Copy the JSON file to assets:**
   ```bash
   cp ../../out/composeOutput.json app/src/main/assets/composeOutput.json
   ```

2. **Open in Android Studio:**
   - Open Android Studio
   - Select "Open an existing project"
   - Navigate to `testing/Android`
   - Click "Open"

3. **Run the app:**
   - Wait for Gradle sync to complete
   - Click the "Run" button (green play icon)
   - Select an emulator or connected device

## Usage

The app automatically loads `composeOutput.json` from the assets folder when it starts. Each component is displayed in a card showing:

- **Component Name**: The generated Compose function name
- **Base Modifiers**: List of all base Compose modifiers
- **States**: Interactive states (hover, pressed, etc.) with their modifiers
- **Responsive**: Media query variants with their modifiers
- **Generated Code**: The complete, ready-to-use `@Composable` function code in a syntax-highlighted code viewer

## Notes

- Make sure to copy the latest `composeOutput.json` to the assets folder before running
- The app requires Android SDK 24 (Android 7.0) or higher
- Uses Jetpack Compose with Material 3
