# iOS SDUI testing environment

Runtime SDUI renderer for the Style Converter — the iOS sibling of
`testing/Android/` and `testing/web/`. Loads `tmpOutput.json` at launch
and renders each component using SwiftUI.

**Primary entry point: [`../../test-all.sh`](../../test-all.sh)** runs
iOS + Android + Web capture + cross-platform comparison. See
[`../README.md`](../README.md) for the full system.

Use this directory's scripts only when you want iOS in isolation:

```bash
# From the repo root:
./test-ios.sh                        # iOS only, skipping Android + Web
./test-ios.sh examples/foo.json      # different input
```

(It's a thin wrapper that runs `SKIP_ANDROID=1 SKIP_WEB=1 ./test-all.sh`.)

## Scope

This is the **basic-rendering scaffold**. Supported today:

| Category   | Properties |
|------------|------------|
| Sizing     | `Width`, `Height`, `Min/MaxWidth`, `Min/MaxHeight`, `AspectRatio` |
| Spacing    | `Padding{Top,Right,Bottom,Left}` (+ logical equivalents) |
| Colors     | `BackgroundColor`, `Color` |
| Borders    | Per-side widths + colors, per-corner radii, uniform stroke |
| Layout     | `Display: flex/grid/inline/none`, `FlexDirection`, `JustifyContent`, `AlignItems`, `FlexWrap`, `Gap` / `RowGap` / `ColumnGap` |
| Typography | `FontSize`, `FontWeight`, `FontStyle`, `LetterSpacing`, `LineHeight`, `TextAlign`, `TextDecoration` |
| Effects    | `Opacity`, `Rotate`, `Scale`, `BoxShadow` (single), `ZIndex` |

Everything else falls through to the default block layout with the
component's name rendered as a placeholder. To add more properties, extend
the `switch prop.type` in
[StyleBuilder.swift](StyleConverterTest/Renderer/StyleBuilder.swift).

## Requirements

- macOS with Xcode 15+ (iOS 16+ simulator)
- [XcodeGen](https://github.com/yonaskolb/XcodeGen) — `test-all.sh` installs
  it via Homebrew automatically if missing.
- `xcpretty` (optional, prettifies build logs): `gem install xcpretty`

## How captures are produced

On first launch, the app:

1. Loads `tmpOutput.json` from the bundle.
2. For every component in the flattened tree, renders it into a
   [`CaptureCanvas`](StyleConverterTest/Screenshot/CaptureCanvas.swift) —
   390 pt wide, natural height, solid `#1A1A2E` background, 16 pt padding.
3. Passes the canvas to SwiftUI's `ImageRenderer` (scale 1.0 to match the
   Android emulator's 160 dpi) and writes each PNG to
   `<Documents>/test_screenshots/{index}_{Name}.png`.
4. `test-all.sh` pulls those out of the simulator sandbox into
   `testing/iOS/screenshots/`.

The chromeless canvas contract is deliberately identical to the Android
and Web capture surfaces — see [`../README.md`](../README.md) for the
cross-platform system and how the comparison report uses it.

## Running from Xcode manually

If you want to iterate from Xcode directly (e.g. using SwiftUI Previews
on renderer changes):

```bash
# 1. Convert + copy IR (or just run test-all.sh once first)
./gradlew run --args="convert --from css --to compose -i examples/visual-test.json -o out"
cp out/tmpOutput.json testing/iOS/StyleConverterTest/Resources/tmpOutput.json

# 2. Generate and open the project
cd testing/iOS
xcodegen generate
open StyleConverterTest.xcodeproj

# 3. ⌘R to run on any iOS 16+ simulator
```

The app captures screenshots on first launch, then shows a browsable
gallery. To re-capture, reinstall the app (or delete its Documents folder).

## Pulling screenshots by hand

```bash
BUNDLE=com.styleconverter.test
UDID=$(xcrun simctl list devices booted | grep -o '[0-9A-F-]\{36\}' | head -1)
CONTAINER=$(xcrun simctl get_app_container "$UDID" "$BUNDLE" data)
open "$CONTAINER/Documents/test_screenshots"
```
