# Style Converter - Web Visual Testing Environment

This is a Next.js-based web application for visually testing CSS component styles generated by the Style Converter.

## Setup

1. Install dependencies:
```bash
npm install
```

2. Run the development server:
```bash
npm run dev
```

3. Open [http://localhost:3000](http://localhost:3000) in your browser

## Usage

### Loading Test Data

Place your CSS component JSON file in the `public/` directory as `test-data.json`.

**Expected JSON format:**
```json
{
  "components": {
    "component-name": {
      "properties": {
        "width": "200px",
        "height": "200px",
        "background-color": "#3b82f6"
      },
      "selectors": [
        {
          "selector": "hover",
          "properties": {
            "background-color": "#2563eb"
          }
        }
      ],
      "media": [
        {
          "query": "max-width: 768px",
          "properties": {
            "width": "100%"
          }
        }
      ]
    }
  }
}
```

- Properties are plain key-value string pairs
- Optional `selectors` array for pseudo-class/pseudo-element styles
- Optional `media` array for responsive media query styles

### Features

- **Visual Preview**: Each component is rendered as a div with its corresponding styles
- **CSS Properties Display**: Raw CSS properties are displayed above each component for reference
- **Centered Titles**: Each component has a centered title showing "Component: {componentName}"
- **Vertical Layout**: Components are displayed one by one vertically for easy comparison

### Testing Workflow

1. Generate CSS output from the Style Converter (Kotlin parser)
2. Copy the output JSON to `public/test-data.json`
3. Refresh the page to see visual results
4. Compare rendered output with expected styles

testing/web/
├── app/
│   ├── globals.css       # Global styles with Tailwind
│   ├── layout.tsx        # Root layout component
│   └── page.tsx          # Main page displaying components
├── public/
│   └── test-data.json    # CSS component test data

├── tsconfig.json
├── next.config.js

## Development

- Edit `app/page.tsx` to modify the component display logic
- Edit `public/test-data.json` to test different CSS properties
- The page automatically reloads when you edit files
